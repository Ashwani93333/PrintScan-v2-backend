# 🖨️ PrintEase (PrintScan v2) - Full-Stack Application

PrintEase is a comprehensive, highly scalable, secure, and production-ready SaaS print shop management platform. It streamlines print job processing, handles large file deliveries via edge CDNs, and provides robust administrative features for shop owners and platform managers.

---

## 🌟 Core Features (Step-by-Step)

### 1. Modern User Experience & Guest Access
*   **Landing Page:** Built with a glass-morphism UI, 3D interactive elements, and a responsive pricing section to engage users immediately.
*   **Direct Shop Access:** Users can navigate directly to a shop's unique URL slug (e.g., `campus-quick-print`) to access their specialized upload portal.
*   **Guest Uploads & Real-Time Pricing:** Users can upload PDFs, JPGs, and PNGs without creating an account. The system natively parses documents (via Apache PDFBox) to calculate page counts and generates an instant price quote based on the shop's specific Color/B&W rates.
*   **Live Tracking:** Upon submission, users receive a unique 6-character tracking token to monitor their job status (Pending, Accepted, Completed) in real-time.

### 2. Shop Owner Administration
*   **Live Dashboard:** Shop admins get real-time queue monitoring to track incoming, pending, and completed print requests.
*   **Job Execution:** Owners can view uploaded files, configure print settings, and update job statuses to notify customers.
*   **Revenue Analytics:** Comprehensive tracking of daily, weekly, and monthly revenue and page counts, aggregated quickly by the backend.
*   **Dynamic QR Codes:** Automatic generation of QR codes (using ZXing) for physical storefronts to enable rapid shop discovery.

### 3. Super Admin Platform Management
*   **Centralized Approval Workflow:** New shop registrations are held in a secure state until manually reviewed and approved by a Super Admin.
*   **Global Metrics & Auditing:** High-level insights into network-wide performance. Caching (Caffeine) ensures that heavy global metric dashboards load instantly.
*   **Access Revocation:** Ability to reject or delete shops, automatically triggering cascading deletions of associated admin user accounts to maintain database integrity.

## 🏗️ Architecture & Strong Full-Stack Integration

PrintEase features a highly optimized separation of concerns designed to handle heavy file uploads and concurrent reads with near-zero latency.

### Lightning-Fast Edge CDN File Delivery
Traditional applications bottleneck when serving large files. PrintEase bypasses this entirely:
1.  **Upload:** The React frontend securely POSTs the user's file to the Spring Boot API.
2.  **Transfer:** Spring Boot streams the file directly to **Cloudflare R2** via the MinIO S3 API client.
3.  **Persist:** The backend saves the **public CDN URL** generated by Cloudflare to the PostgreSQL database.
4.  **Fetch (Zero Server Load):** When a shop owner or user views a file, the React frontend loads it directly from Cloudflare's global edge network using the CDN URL. The Spring Boot backend is completely bypassed for reads, resulting in instant delivery and zero compute load/egress costs.

## 🔐 Enterprise-Grade Security

Security is deeply integrated into both the frontend state management and backend API architecture.

*   **Stateless Authentication (JWT):** We use **JJWT** for robust token generation. Crucially, tokens are stored in `HttpOnly`, `Secure`, and `SameSite` cookies. This strictly isolates the tokens from client-side JavaScript, effectively neutralizing Cross-Site Scripting (XSS) attacks (no `localStorage` usage).
*   **Google OAuth 2.0 Integration:** Seamless Single Sign-On (SSO) integrated alongside traditional email/password credentials.
*   **Robust CSRF Protection:** The frontend initiates a request to an `/api/auth/csrf` endpoint. Axios interceptors automatically attach the `X-XSRF-TOKEN` to all mutating requests (POST, PUT, PATCH, DELETE). The Spring Boot backend synchronizes and validates this via `CookieCsrfTokenRepository` to prevent Cross-Site Request Forgery.
*   **Role-Based Access Control (RBAC):** Strict JWT claims map to `ROLE_USER`, `ROLE_ADMIN` (shop owners), and `ROLE_SUPERADMIN`. These roles aggressively protect backend API endpoints and are mirrored on the frontend to guard React routes and trigger secure redirects.

## 🚀 Deployment Cycle

The full-stack application is battle-tested and optimized for modern PaaS and serverless deployments.

### 1. Frontend Compute (Vercel)
*   **Continuous Deployment:** Connected directly to GitHub, triggering optimized Vite builds (`npm run build`) on every push.
*   **Routing Configuration:** Vercel inherently manages SPA routing fallbacks.
*   **Configuration:** Environment variables (`VITE_API_BASE_URL`) point directly to the production API gateway.

### 2. Backend Compute (Railway / Render)
*   **Dockerized Deployment:** Deployed seamlessly via the provided `Dockerfile`.
*   **Stateless & Scalable:** Because session state is JWT-based and database-independent, the API can scale horizontally automatically under heavy traffic.
*   **SSL Termination:** Encrypted traffic management handled natively by the PaaS provider.

### 3. Database (Neon Serverless Postgres)
*   Fully managed PostgreSQL with decoupled compute and storage.
*   Connection pooling configured for high concurrent user sessions and automatic scaling capabilities.

### 4. Object Storage (Cloudflare R2)
*   Configured with strict CORS policies to allow cross-origin reads exclusively from the verified frontend domain.
*   Zero egress fees make it the perfect solution for a file-heavy SaaS platform.

## 💻 Local Development Setup

### Frontend Setup
1.  Navigate to the frontend directory: `cd printScan-v2`
2.  Install dependencies: `npm install`
3.  Configure environment: Create a `.env` file with `VITE_API_BASE_URL=http://localhost:8080`
4.  Run development server: `npm run dev`

### Backend Setup
1.  Navigate to the backend directory: `cd printScan-v2-backend`
2.  Configure Database: Ensure PostgreSQL is running. Update `src/main/resources/application-dev.yml` with your local DB credentials.
3.  Run the application using Maven: `./mvnw spring-boot:run`
4.  API Documentation: Access Swagger UI at `http://localhost:8080/swagger-ui.html`
